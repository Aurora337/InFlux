# InFlux Code of Conduct

## Our Commitment

The InFlux project is committed to fostering a professional, respectful, and collaborative environment for everyone who participates in the development of the protocol.

We welcome contributions from developers, researchers, validators, auditors, and community members regardless of experience level.

---

## Expected Behavior

Participants are expected to:

* Treat others with professionalism and respect.
* Focus discussions on technical improvement.
* Welcome constructive feedback.
* Share knowledge openly.
* Help maintain accurate documentation.
* Encourage collaboration.

---

## Unacceptable Behavior

The following behavior is not acceptable:

* Harassment or discrimination.
* Personal attacks.
* Threats or intimidation.
* Spam or malicious contributions.
* Deliberately disruptive behavior.
* Knowingly submitting malicious code.

---

## Technical Discussions

Disagreements are expected during protocol development.

Technical decisions should be based on:

* Determinism
* Security
* Performance
* Maintainability
* Reproducibility

---

## Enforcement

Project maintainers reserve the right to remove contributions or restrict participation when behavior violates this Code of Conduct.

---

## Reporting Issues

If you experience or witness behavior that violates this Code of Conduct, please report it to the project maintainers with as much detail as possible.

Reports will be handled respectfully and confidentially whenever practical.

---

## Scope

This Code of Conduct applies to:

* GitHub Issues
* Pull Requests
* Discussions
* Documentation
* Community spaces
* Project communications

---

## Our Goal

The long-term goal of InFlux is to build a secure, deterministic, and professionally maintained protocol through open collaboration and mutual respect.
