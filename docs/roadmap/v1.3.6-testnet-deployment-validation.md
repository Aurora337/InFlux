# v1.3.6 Testnet Deployment Validation

## Goal
Validate that testnet deployment paths are reproducible, deterministic, and promotion-ready across local/dev container and CI contexts.

## Scope
In scope:
- Deterministic deployment preflight checks
- Deployment validation script and report contract
- CI gate for deployment validation
- Failure-mode checks for missing config/artifacts

Out of scope:
- New protocol logic
- Feature expansion unrelated to deployment readiness
- Runtime-generated report baselines committed by default

## Slice 1 Checklist (Deployment Preflight)

### Deliverables
- [ ] Add preflight validator script: scripts/testnet/validate_testnet_deployment_preflight.py
- [ ] Emit structured JSON report: docs/testnet/deployment_preflight_report.json
- [ ] Add tests for output schema and deterministic behavior
- [ ] Add failure tests for missing/invalid deployment prerequisites
- [ ] Add CI step to run preflight validator in pull requests

### Required Checks (Preflight)
- [ ] Required directories exist (testnet/configs, testnet/genesis, testnet/validators)
- [ ] Genesis/config artifacts parse as valid JSON where applicable
- [ ] Validator config count satisfies minimum threshold
- [ ] No duplicate validator identity entries
- [ ] Required launch scripts are present and executable
- [ ] Deterministic hash/fingerprint generated for preflight inputs

### Report Contract (Initial)
- [ ] preflight_valid: bool
- [ ] preflight_score: float in [0, 1]
- [ ] checks_passed: int
- [ ] checks_total: int
- [ ] input_fingerprint: string
- [ ] timestamp: RFC3339 UTC string
- [ ] checks: object with per-check valid/message fields

### Acceptance Criteria
- [ ] Preflight script exits 0 when all checks pass, non-zero otherwise
- [ ] Two consecutive runs with identical inputs produce identical non-timestamp fields
- [ ] CI passes on PR with deterministic validation enabled
- [ ] Runtime-generated reports are restored/ignored after validation runs (not committed by default)
