# v1.3.7 - Deterministic Cluster Formation Specification (DCFS)

## Purpose

Define how nodes self-organize into stable clusters under event pressure without introducing randomness.
This slice is the system emergence layer between kernel execution and large-scale testnet topology behavior.

## Deterministic Inputs

Cluster formation is computed only from:

1. VPU density
2. Alignment coherence over the P/B/FI/PT vector
3. Temporal event clustering via CTOR window overlap
4. Node type convergence across VN/SN/REN/PTN

No random seeding, probabilistic tie-breaking, or external oracle input is allowed.

## Genesis Condition

A cluster C_i is created only when all conditions hold inside a CTOR window:

1. sum(VPU) >= tau_cluster
2. alignment_entropy <= epsilon
3. CTOR ordering is stable

Where:

1. tau_cluster is the minimum aggregate pressure threshold
2. epsilon is low deterministic chaos tolerance

## Cluster Classes

1. Verification Cluster (VC): VN-heavy trust anchoring and validation throughput
2. Economic Cluster (EC): REN-heavy reproduction zones
3. Platform Cluster (PC): PTN-heavy routing and platform throughput
4. Hybrid Stability Cluster (HSC): mixed VN/SN/REN long-horizon equilibrium

## Cluster State Equation

Each cluster state is computed as:

S(C) = f(VPU_density, alignment_coherence, node_entropy, reserve_pressure)

The implementation uses deterministic weighted composition:

state_score = 0.35 * vpu_density + 0.35 * alignment_coherence + 0.20 * (1 - node_entropy) + 0.10 * (1 - reserve_pressure)

State score is clamped to [0, 1].

## Lifecycle States

1. stable
2. merging
3. splitting
4. decaying

## Transition Rules

### Merge

Two clusters may merge when:

1. Alignment overlap >= merge_alignment_threshold
2. VPU stream similarity >= merge_vpu_similarity_threshold

### Split

A cluster must split when either condition holds:

1. node_entropy > split_entropy_max
2. CTOR ordering stability is false

### Decay

A cluster decays when either condition holds:

1. inactivity_windows >= decay_inactivity_windows
2. ren_activity_cycles == 0 and vpu_density <= decay_min_vpu_density

## Emergence Pipeline

Event Stream
-> VNode observation
-> VPU aggregation
-> Alignment mapping
-> CTOR windowing
-> Cluster detection engine
-> Cluster formation/update

## Safety Constraints

1. No cluster may self-instantiate without event history.
2. No cluster may exist outside CTOR ordering rules.
3. Cluster state is observational and cannot override ledger state.
