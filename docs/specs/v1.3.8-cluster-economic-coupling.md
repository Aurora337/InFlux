# v1.3.8 - Cluster Economic Coupling (CEC)

## Purpose

Establish a deterministic observation-only coupling layer between cluster emergence and economic analytics.

Clusters remain non-authoritative for state mutation.
Economic reproduction and supply controls remain bound to VPU and ledger rules already defined by the economic engine.

## Scope

The Cluster Economic Coupling Engine (CECE) computes deterministic metrics for each cluster:

1. demand_pressure
2. reserve_pressure
3. reproduction_pressure
4. stability_index

Outputs are analytics-only and cannot directly modify:

1. supply
2. reserve balances
3. ledger contents
4. consensus ordering

## Deterministic Requirements

For identical cluster input snapshots, every node class (VN, SN, REN, PTN, AN) must produce identical metric outputs.

No randomness, no wall-clock variance, and no external service dependency are permitted.

## CECE Inputs

Per cluster snapshot:

1. cluster_id
2. vpu_density
3. alignment_coherence
4. event_intensity
5. reserve_utilization
6. reserve_drift
7. cluster_entropy
8. role_composition (VN/SN/REN/PTN/AN)

## CECE Outputs

ClusterMetrics:

1. demand_pressure
2. reserve_pressure
3. reproduction_pressure
4. stability_index
5. role_composition_normalized

All outputs are bounded to [0, 1].

## Deterministic Coupling Formulation

### Demand Pressure

demand_pressure = clamp(0.50 * vpu_norm + 0.30 * alignment_coherence + 0.20 * event_intensity)

### Reserve Pressure

reserve_pressure = clamp(0.70 * reserve_utilization + 0.30 * reserve_drift)

### Reproduction Pressure

reproduction_pressure = clamp(0.50 * vpu_norm + 0.30 * ren_share + 0.20 * demand_pressure)

### Stability Index

stability_index = clamp(
0.40 * alignment_coherence
+ 0.25 * (1 - reserve_pressure)
+ 0.20 * (1 - abs(demand_pressure - reproduction_pressure))
+ 0.15 * (1 - cluster_entropy)
)

## Continuity Rules

### Merge Continuity

When clusters merge, aggregate metrics are computed by deterministic weighted averaging.

### Split Continuity

When a cluster splits, child metrics are deterministically projected from parent metrics by fixed split ratios.

Both operations preserve metric continuity under replay.
