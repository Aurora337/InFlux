# InFlux v0.1 Release Notes

## Purpose
Establish a deterministic kernel baseline that can execute repeatably from a defined genesis state.

## Features Added
- Core state model and deterministic state transition pipeline.
- Canonical state serialization and root hash computation.
- Runtime entrypoint for executing kernel steps end to end.

## Validation Results
- Deterministic execution confirmed with stable hash outputs for repeated runs.
- Kernel audit artifact established as the baseline reference.

## Known Limitations
- No replay pipeline for historical verification yet.
- No multi-node consensus behavior yet.
- No resilience or fault-injection coverage yet.

## Next Milestone
v0.2 Replay Verification: prove deterministic replay and ledger integrity from persisted history.
