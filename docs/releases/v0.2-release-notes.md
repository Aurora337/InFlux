# InFlux v0.2 Release Notes

## Purpose
Prove that historical execution can be replayed deterministically and verified against stored ledger history.

## Features Added
- Replay runner, replay store, replay report, and replay audit workflows.
- Ledger verification CLI and scenario-driven replay execution.
- Replay metrics for pass/fail and determinism scoring.

## Validation Results
- Replay scenario aggregate passed with deterministic results and full ledger integrity.
- Reported replay success rate and determinism score reached 1.000 for validated suites.

## Known Limitations
- Consensus behavior across multiple validators not yet validated.
- Fault recovery behavior under adverse network conditions not yet tested.

## Next Milestone
v0.3 Consensus Validation: validate multi-node agreement and divergence tracking.
