# Documentation Standards

All documentation should:

* Use deterministic terminology
* Maintain consistent formatting
* Include version information
* Remain synchronized with implementation
* Be reviewed before release

Documentation follows the same lifecycle as protocol development:

1. Draft
2. Review
3. Approved
4. Released
5. Maintained
6. Archived
