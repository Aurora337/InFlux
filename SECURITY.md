# Security Policy

## Supported Versions

Security updates are provided for the most recent development branch and all officially released production versions.

---

## Reporting a Vulnerability

If you discover a security vulnerability within the InFlux protocol, please report it privately to the project maintainers.

When reporting an issue, include:

* A description of the vulnerability.
* Steps to reproduce.
* Potential impact.
* Suggested mitigation (if known).
* Relevant logs or screenshots.

Please avoid publicly disclosing vulnerabilities until they have been investigated and addressed.

---

## Security Principles

The InFlux protocol is designed around:

* Deterministic execution
* Consensus integrity
* Secure validator communication
* State consistency
* Replay protection
* Economic integrity

Security is considered a core feature of the protocol rather than an optional enhancement.

---

## Responsible Disclosure

Researchers who responsibly disclose vulnerabilities help improve the long-term security of the protocol.

Contributors are encouraged to coordinate with maintainers before publishing security-related findings.

---

## Scope

This policy applies to:

* Protocol implementation
* Consensus engine
* Economic engine
* Validator lifecycle
* Cluster formation
* State replication
* Network communication
* Governance components

---

## Security Updates

Security fixes are documented in release notes and incorporated into future protocol versions after validation and testing.
